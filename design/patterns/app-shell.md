# 应用外壳模式（App Shell）

> 数值见 `tokens/primitives.json → breakpoints / layoutMetrics`。

## 窗口分级

| 级别 | 条件 | 壳形态 |
|---|---|---|
| compact | 移动：宽 ≤600 或 高 <600 | 底栏 |
| medium | 桌面：宽 <1100；移动：宽 <1000 | 侧栏（桌面）/ 底栏（移动） |
| wide | 桌面：宽 ≥1100；移动：宽 ≥1000 | 侧栏 |

**移动壳判定**：非桌面平台且（宽 <820 或 高 <600）。桌面平台任何尺寸都保持桌面信息架构——窗口临时变矮不得退化成手机导航。

## 桌面壳

```
┌──────────┬───────────────────────────────┐
│ 侧栏      │ 内容（SafeArea 顶=标题栏高度）  │
│ chromeSurface │                            │
│ 全高      │                               │
└──────────┴───────────────────────────────┘
  标题栏透明叠加在最上方（不占布局）
```

- 侧栏全高延伸到标题栏下方；macOS 内容避让交通灯（38pt  inset，阅读器等自定义 chrome 避让 78）；
- Windows：自绘标题栏 44px（拖动区 + 最小化/最大化/关闭 40×32 按钮，关闭 hover #E81123@90%）；
- 标题栏透明，下方内容延续（侧栏/画布透上来）。

## 移动壳

- 底栏常驻（见 `components/navigation.md`）；`Scaffold.extendBody` 让内容延伸到底栏玻璃下；
- 内容底部留白：移动壳 140 / 桌面 96（保证末行不被遮挡）。

## 页面级布局

- 页边距按窗口级别：16 / 24 / 32；
- 页标题 26/28 w800 负字距，左对齐（AppBar 不居中）；
- AppBar：透明/画布色、无 elevation、无 scrolledUnder 色。

## 规则

1. 壳切换只由窗口分级驱动，产品不得自造断点；
2. 顶层目的地 3–5 个；二级页面进栈（返回钮）不占导航；
3. 品牌名只出现在侧栏顶部 / 标题栏，不重复出现在内容页头。
